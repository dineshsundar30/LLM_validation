*****************  DeepEval — The Framework  ******************************

DeepEval is an open-source Python framework for evaluating LLM applications. It provides five core capabilities:

📏 Metrics
    Pre-built evaluation criteria.
    Examples: Answer Relevancy, Faithfulness, Tool Correctness.
    Supports custom criteria via GEval.

📂 Test Cases (Goldens)
    Structured containers for:
    Inputs
    Outputs
    Expected values

⚖️ Judge LLM
    An LLM that scores the outputs.
    It is a separate model from the one being tested.

👁️ Tracing / Instrumentation
    Hooks into LangChain, OpenAI, etc.
    Automatically captures:
    Tool calls
    Intermediate steps
    Retrieved documents

▶️ Test Runner
    Runs evaluation tests.
    Integrates with pytest or standalone methods:
    evaluate()
    evals_iterator()


## How Scoring Works in deepeval

* Score Range: 0.0 – 1.0
* Threshold = Pass/Fail line

### Score Levels

* 1.0 (Perfect): Fully satisfies metric
* 0.7–0.9 (Good): Minor issues, overall solid
* 0.4–0.6 (Mediocre): Needs improvement
* 0.1–0.3 (Poor): Major failures

### Thresholds

* Quality Metrics: 0.7 → AnswerRelevancy, TaskCompletion, GEval
* Safety Metrics: 0.5 → Bias, Toxicity, PII
* Efficiency Metrics: 0.5 → StepEfficiency

Recall: Score (0–1) + Threshold = Pass/Fail
Score >= Threshold → ✅ Pass / Good
Score < Threshold → ❌ Fail / Needs improvement
